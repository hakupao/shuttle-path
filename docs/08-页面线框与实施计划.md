## 一、全局组件改版

### 1.1 导航栏（Nav）

**现状问题：**
- 衬线体品牌名 → 书卷气
- 圆形 Logo 按钮 → 太常规
- 玻璃态毛玻璃背景 → 偏模板感

**改版方案：**

```
桌面端：
┌──────────────────────────────────────────────────────────────┐
│  🏸 Shuttle Path          首页    知识库    教学日记          │
│  (得意黑,倾斜)            (Noto Sans SC 500, hover 变粗)     │
└──────────────────────────────────────────────────────────────┘

移动端：
┌────────────────────────────────┐
│  🏸 Shuttle Path          ☰   │
└────────────────────────────────┘
展开：
┌────────────────────────────────┐
│  🏸 Shuttle Path          ✕   │
├────────────────────────────────┤
│  首页                          │
│  知识库                        │
│  教学日记                      │
└────────────────────────────────┘
```

**关键变化：**
| 元素 | 现在 | 改为 |
|------|------|------|
| 品牌字体 | Instrument Serif（衬线） | 得意黑（窄体倾斜无衬线） |
| Logo 形状 | 圆形绿底 | 手绘风羽毛球 SVG 小图标（无背景色块） |
| 背景 | 毛玻璃 `backdrop-blur` | 纯白 + 底部 1px 浅灰线（干净利落）|
| 导航链接 hover | 下划线从左到右生长 | **字重从 400→700 渐变**（variable font 思路），底部加 2px 活力绿色条 |
| 间距 | 紧凑 h-16 | 保持 h-16，但链接间距加大，呼吸感 |
| 移动菜单动画 | max-height 展开 | 从上方滑下 + 弹性缓动 |

### 1.2 页脚（Footer）

**现状问题：**
- 分隔线 + 羽毛球图标居中 → 太对称、太模板
- 双列链接结构 → 链接太少不需要这么正式

**改版方案：**

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  ~~~~~~~~~~~~~ 手绘波浪线分隔 ~~~~~~~~~~~~~                  │
│                                                              │
│  Shuttle Path                     知识库 · 教学日记 · GitHub │
│  (得意黑)                         (一行排列,点号分隔)        │
│                                                              │
│           一个羽毛球爱好者的教学笔记与知识库                  │
│                                                              │
│  © 2026 · Made with ♥ by hakupao                             │
│                                                              │
│           [散落的羽毛球手绘装饰碎片，半透明]                  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**关键变化：**
| 元素 | 现在 | 改为 |
|------|------|------|
| 分隔线 | 直线 + 居中图标 | SVG 手绘波浪线 |
| 布局 | 左右两列 | 单列居中，更简洁 |
| 链接排列 | 分组双列 | 一行用 `·` 分隔 |
| 装饰 | 无 | 角落散落几个手绘羽毛球碎片（半透明，纯装饰） |
| 背景 | `paper-warm/40` | 极浅灰 `#F9FAFB`，和主体内容区分开 |

### 2.2 Hero 区域设计细节

**与现在的核心区别：**

| 维度 | 现在 | 改为 |
|------|------|------|
| 背景 | 抽象球场线（极淡描边矩形） | **无几何背景**，改用右侧手绘大插画 + 纸质噪点纹理 |
| 标题字体 | `font-brand`（Instrument Serif） | **得意黑**，`clamp(3rem, 7vw, 5.5rem)` |
| 标题颜色 | `court-dark` 墨绿 | **石墨 `#1F2937`**，干净现代 |
| 强调线 | 12px × 2px 琥珀色线条 | **去掉**，让标题自身的字体力量说话 |
| 按钮形状 | `rounded-lg`（标准圆角） | **超椭圆** |
| 按钮颜色 | 墨绿 `court` | **活力绿 `#10B981`** |
| 动效 | fade-in-up 700ms | **fade-in-up 400ms** + 手绘插画的路径描绘动画 |
| 副标题 | "羽毛球教学知识平台" 16px | 保持，但加大到 **20px**，字重 **500** |

**Hero 右侧插画构想：**
- 一只手绘风格的羽毛球，正在飞行中
- 身后拖着一条手绘弧线轨迹（虚线 + 实线交替）
- 弧线末端散出几根羽毛碎片
- 整个插画用 SVG，线条色 `#3f3a36`（墨线色）
- 填充用半透明的绿和橙水彩色块
- 页面加载时，轨迹线有描绘生长动画（SVG stroke-dashoffset）

### 2.3 知识库分类 — Bento 布局

**现在是三列等宽等高，改为 Bento 不等比：**

```css
/* Bento Grid */
.knowledge-bento {
  display: grid;
  grid-template-columns: 1.2fr 0.8fr;
  grid-template-rows: auto auto;
  gap: 20px;
}

/* 手法（大卡片，占左列两行） */
.bento-strokes {
  grid-row: 1 / 3;
}

/* 步法（右上） */
.bento-footwork {
  grid-row: 1;
}

/* 运动科学（右下） */
.bento-science {
  grid-row: 2;
}
```

**移动端退化为单列竖排。**

**卡片设计：**
- 超椭圆（非标准 border-radius）
- 表面有极淡噪点纹理
- 图标改为手绘风 SVG（替代当前的线条图标）
- hover：整体上浮 4px + 阴影加深 + 图标区域背景变为活力绿
- 大卡片内可以放更丰富的内容（小文章预览列表）

### 2.4 教学日记卡片

**与现在的区别：**

| 维度 | 现在 | 改为 |
|------|------|------|
| 顶部 | 纯色色块 h-28 + 点阵纹理 | **去掉色块**，改为卡片整体有左侧 4px 彩色竖条（手绘风不规则粗细）|
| 旋转 | 无 | **微旋转** `rotate(-0.8deg)` ~ `rotate(1.2deg)` |
| hover | 阴影加深 | **归正旋转** + 上浮 + 阴影 |
| 日期 | 嵌在色块里的徽章 | 卡片内顶部，小字 |
| 标签 | `bg-paper-warm` 圆角胶囊 | **手绘风下划线标签**（不是背景色块，而是文字 + 手绘下划线） |

## 七、动效规格

### 7.1 全局缓动函数

```css
/* 现在 */
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);

/* 新增 */
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);  /* 弹性，用于 hover */
--ease-smooth: cubic-bezier(0.25, 0.1, 0.25, 1);    /* 平滑，用于页面过渡 */
```

### 7.2 页面入场动画

```
现在：fade-in-up 700ms（所有元素统一）

改为：
- 标题：fade-in-up 400ms（快速到位）
- 副文字：fade-in-up 400ms，延迟 100ms

```

## 十、需要确认的问题

在开始实施前，有几个需要你拿主意的点：

1. **超椭圆兼容性**：CSS `corner-shape: squircle` 目前只有 Chrome 139+ 支持。
   - 方案 A：用 CSS 原生 + `@supports` 回退（简单，但 Safari/Firefox 看到的还是普通圆角）
   - 方案 B：用 JS 库 `squircle-js`（全浏览器一致，但需要引入依赖）
   - 方案 C：对关键组件用 SVG clip-path（手动但精确）
   - **建议：方案 A，渐进增强，不为这个引入额外 JS**

2. **手绘图标来源**：
   - 方案 A：你自己画 → 最有"手感"
   - 方案 B：AI 生成草图 + 手工微调 → 效率高
   - 方案 C：用 Rough.js 代码生成 → 零人工，但可控性低
   - **建议：核心图标用 B，装饰元素用 C**

3. **得意黑字体加载策略**：
   - 完整 woff2 大约 4-6MB（含完整中文字符集）
   - 只用于标题，可以考虑子集化（subset）只保留常用字 → 大幅减小到 500KB 以内
   - **建议：先全量加载 + `font-display: swap`，后续按需子集化优化**

4. **知识库分类如果以后超过 3 个**，Bento 布局怎么扩展？
   - 可以预留一套 4 分类、5 分类的 grid-template 变体
   - 或者超过 3 个后退化为等宽网格
   - **建议：先做 3 分类的 Bento，后续按需调整**
