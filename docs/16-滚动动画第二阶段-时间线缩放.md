# 16 - 滚动动画第二阶段：转播视角 + 镜头升降

## 核心概念

把网页视口当作羽毛球转播摄像机，球场从正俯视（90°鸟瞰）改为转播视角（约 25°~35° 俯角），镜头跟随羽毛球在球场上空反复**俯冲聚焦、拉起飞行**。

## 确认的设计决策

| 项目 | 决策 |
|------|------|
| 球场透视 | **完整 CSS 3D perspective** — 真正的近大远小梯形变形 |
| 文字浮空 | **干净全息** — 柔和白/蓝光晕 + 微弱投影，像体育馆记分牌的高级版 |
| 聚焦放大 | **充满视口** — 聚焦时内容占满整个屏幕 |
| 升空过渡 | **完整球场全景** — 拉起时能看到整个球场和所有模块 |
| 视角动态 | **俯角动态变化** — 聚焦时俯角变大（更俯视），升空时俯角变小（更平视） |
| 移动端 | **完整保留** — 手机也做完整 3D 透视 + 升降循环 |
| 性能底线 | **流畅优先** — 60fps，必要时降级效果 |
| 内容区域 | **保持现有** — 课程 → 知识库 → 分类，3 个着陆点不变 |

## 滚动体验流程

```
[页面加载]
  球场以转播视角（~30° 俯角）呈现，羽毛球入场
    ↓ 滚动
[聚焦：Hero]
  镜头俯冲，俯角增大，Hero 内容充满视口
  全息文字清晰可读，浮在球场上方
    ↓ 滚动
[升空过渡]
  镜头拉起，俯角减小（更平视），看到球场全景
  羽毛球飞行中，内容缩小退后
    ↓ 滚动
[聚焦：课程区]
  镜头再次俯冲聚焦，课程区充满视口
    ↓ 滚动
[升空过渡]
  ...同上循环...
    ↓ 滚动
[聚焦：知识库]
    ↓ 滚动
[升空过渡]
    ↓ 滚动
[聚焦：分类区]
    ↓ 滚动
[结束动画]
  羽毛球退场，镜头稳定
```

## 视觉参考

### 球场透视
- BWF 标准转播主机位：球场一端后方高处，25°~35° 俯角
- CSS `perspective` + `rotateX` 实现梯形变形
- 聚焦时俯角增大（如 45°~50°），更接近鸟瞰
- 升空时俯角减小（如 20°~25°），更有纵深感

### 全息文字
- 柔和的白色/浅蓝色光晕（text-shadow / drop-shadow）
- 微弱的地面投影暗示浮空高度
- 不加扫描线/闪烁/故障等重特效
- 保持可读性为第一优先级

### 性能策略
- 优先使用 GPU 加速属性（transform, opacity）
- 60fps 不达标时逐步降级：减少阴影层数 → 简化投影 → 降低透视复杂度
- `prefers-reduced-motion` 用户直接跳过 3D 效果
