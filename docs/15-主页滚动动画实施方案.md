| 项目 | 决定 |

## 四、羽毛球 SVG 设计

俯视视角的羽毛球简笔画：

```
   │ │   │ │    ← 羽毛（扇形展开）
    ╲ ╲ ╱ ╱
      ╲ ╱
       ●          ← 球托（圆形）
```

- 颜色：白色填充 + 白色描边，微透明（适配绿色球场背景）
- 尺寸：约 24×36px（移动端），32×48px（桌面端）
- 包含阴影层：同形状，黑色半透明，`filter: blur()`

## 五、动画时间线设计

### 5.1 入场动画（页面加载，CSS keyframe，非滚动驱动）

```
0ms        → 球在画面上方不可见
0~600ms    → 球从上方飞入，落到 Hero 标题右侧
600~800ms  → 轻微弹跳/缓停
```

入场完成后，交给 ScrollTrigger 接管。

### 5.2 滚动动画时间线

滚动范围：从球场顶部到底部（`.court` 元素的完整高度）。

```
滚动进度    球的位置        球的状态      内容状态
─────────────────────────────────────────────────
0%         Hero 区域       清晰/大       Hero 正常显示
           （起点）        阴影远

0~15%      ↓ 飞向涨球记录  清晰          涨球记录模糊
           略向右偏移

15~25%     涨球记录区域    模糊/小       涨球记录清晰
           悬停在内容上方   阴影近        ← 景深切换

25~40%     ↓ 飞越球网      清晰          知识库模糊
           经过球场中线     scale 最大    ← 弧线最高点

40~55%     知识库区域      模糊/小       知识库清晰
           悬停在内容上方   阴影近        ← 景深切换

55~70%     ↓ 飞向手法/步法  清晰          手法/步法模糊

70~85%     手法/步法区域   模糊/小       手法/步法清晰
                                        ← 景深切换

85~100%    ↓ 飞向底部      清晰渐隐      运动科学正常
           逐渐消失
```

### 5.3 景深切换参数

**飞行中（球聚焦）：**
```css
.shuttlecock { filter: blur(0); opacity: 1; transform: scale(1.2); }
.court-section { filter: blur(3px); opacity: 0.7; }
.shuttle-shadow { filter: blur(8px); offset-y: 12px; }
```

**到达区域（内容聚焦）：**
```css
.shuttlecock { filter: blur(3px); opacity: 0.6; transform: scale(0.8); }
.court-section { filter: blur(0); opacity: 1; }
.shuttle-shadow { filter: blur(3px); offset-y: 4px; }
```

### 5.4 飞行路径（俯视角度）

球不是直线飞，而是略带左右摆动的曲线，模拟高远球的水平偏移：

```
     ●              ← Hero（中偏左）
      \
       \
        ●            ← 涨球记录（中偏右）
       /
      /
═════●═══════        ← 球网（正中）← 弧线最高点
      \
       \
        ●            ← 知识库（中偏右）
       /
      /
     ●              ← 手法/步法（中偏左）
      \
       ●             ← 底部消失
```

用 GSAP MotionPath 或手动 keyframe 控制 `left` 百分比实现。

## 六、技术实现要点

### 6.1 元素结构

```html
<!-- 在 .court 内部，z-index 最高 -->
<div class="shuttle-container">
  <div class="shuttle-shadow"><!-- SVG 阴影 --></div>
  <div class="shuttlecock"><!-- SVG 羽毛球 --></div>
</div>
```

- `.shuttle-container`：`position: absolute`，在 `.court` 内定位
- GSAP 控制其 `top`（Y 轴位移）、`left`（X 轴摆动）、`scale`、`filter`

### 6.2 ScrollTrigger 配置

```js
import { gsap } from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';
gsap.registerPlugin(ScrollTrigger);

// 主时间线，绑定到 .court 的滚动
const tl = gsap.timeline({
  scrollTrigger: {
    trigger: '.court',
    start: 'top top',
    end: 'bottom bottom',
    scrub: 1,           // 平滑跟随滚动，1秒缓冲
  }
});

// 依次添加各段动画...
tl.to('.shuttlecock', { top: '20%', left: '55%', scale: 1.2, ... })
  .to('.shuttlecock', { top: '25%', scale: 0.8, filter: 'blur(3px)', ... })
  // ...
```

### 6.3 内容区模糊处理

给每个内容区域加 `data-section` 属性，GSAP 在对应滚动段控制其 `filter` 和 `opacity`：

```html
<div class="court-row court-row-lessons" data-section="lessons">
```

### 6.4 性能考虑

- `will-change: transform, filter` 提前声明，触发 GPU 加速
- `filter: blur()` 在移动端有性能开销，限制模糊值不超过 4px
- ScrollTrigger 的 `scrub` 自带 debounce，不会每帧都计算
- 入场动画用 CSS keyframe（不依赖 JS 加载完成）

### 6.5 降级策略

如果浏览器不支持或 JS 加载失败：
- 羽毛球不显示
- 内容区保持正常显示（无模糊）
- 现有的 `fade-in-up` 动画继续生效
