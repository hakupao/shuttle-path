# 18 — 滚动动画微调优化方案

## 问题

1. **滚动距离过长**：700vh scene → ~51 次滚轮到底，4 个内容区各占 ~9 次，用户早已看完内容仍在等镜头缓动
2. **逐帧闪烁/抖动**：每滚动一格滚轮，画面出现明显抖动

## 抖动根因分析

| 原因 | 影响 | 严重度 |
|------|------|--------|
| 球和聚光灯用 `top`/`left` 做位置动画 | 每帧触发 Layout Reflow（最昂贵的渲染阶段） | ★★★ |
| `.shuttle-svg` 的 `filter: drop-shadow()` | opacity 变化时每帧重新渲染滤镜 | ★★ |
| `.shuttle-shadow` 的 `filter: blur()` | transform 动画叠加 filter 导致额外重绘 | ★ |

### 核心问题：`top`/`left` 触发 Layout Reflow

浏览器渲染管线：

```
JS → Style → Layout → Paint → Composite
```

- `transform`（x/y/scale/rotate）→ 只触发 **Composite**（GPU 直接处理，极快）
- `opacity` → 只触发 **Composite**
- `top`/`left` → 触发 **Layout → Paint → Composite**（全链路回流，极慢）

当前代码对球和聚光灯使用 `top`/`left` 做位移动画，每帧触发完整的 Layout Reflow。在 3D `preserve-3d` 上下文中，Reflow 还会波及整个 camera-rig 子树，导致明显卡顿。

## 优化方案

### A. 缩短滚动距离（51次 → ~35次）

| 参数 | 当前 | 优化后 | 变化 |
|------|------|--------|------|
| `.scene` 高度 | 700vh | **500vh** | -200vh |
| 有效滚动距离 | 600vh | **400vh** | -33% |
| 聚焦段 dur | 5 | **3.5** | -30% |
| 过渡段 dur | 2.5 | **2** | -20% |
| 退出段 dur | 2 | **1.5** | -25% |
| Timeline 总时长 | 29.5 | **21.5** | -27% |

### B. 消除 Layout Thrashing

将球和聚光灯的位移从 CSS 布局属性改为 GPU 合成属性：

```js
// Before — 触发 Layout Reflow
tl.to(shuttle,    { top: ty, left: sx + '%', ... });
tl.to(spotlightEl, { top: ty, left: sx + '%', ... });

// After — 纯 GPU Composite
tl.to(shuttle,    { y: ty, x: courtX(sx), ... });
tl.to(spotlightEl, { y: ty, x: courtX(sx), ... });
```

CSS 配合修改：
- `.shuttle`：`top: 0; left: 0;`（位置完全交给 GSAP transform）
- `.spotlight`：同上

### C. 移除昂贵的 CSS Filter

| 元素 | 原 filter | 替代方案 |
|------|----------|---------|
| `.shuttle-svg` | `drop-shadow(0 0 6px rgba(255,255,255,0.5))` | 移除（后续可用独立图层补回） |
| `.shuttle-shadow` | `blur(3px)` | `radial-gradient` 模拟柔化边缘 |

### D. 增加 scrub 缓冲

`scrub: 2` → `scrub: 2.5`（增加平滑度，吸收离散滚轮输入的冲击感）
