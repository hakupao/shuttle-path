# 内容更新教程

本文档介绍如何为 Shuttle Path 添加和更新内容。

---

## 1. 项目结构速览

```
src/content/
├── knowledge/          ← 知识库文章 (MDX)
│   ├── grip.mdx
│   ├── basic-stance.mdx
│   └── clear.mdx
└── lessons/            ← 教学日记 (MDX)
    └── spring-footwork.mdx
```

- **知识库**：手法 / 步法 / 运动科学，统一学术排版
- **教学日记**：每节课一篇，视觉风格完全自由

---

## 2. 添加知识库文章

### 步骤

1. 在 `src/content/knowledge/` 下新建 `.mdx` 文件
2. 文件名使用 **kebab-case**（如 `drop-shot.mdx`）
3. 填写 frontmatter（必填项见下方）
4. 使用标准 Markdown 语法编写内容

### Frontmatter 模板

```yaml
---
title: "文章标题"
description: "一句话描述"
category: "strokes"          # strokes | footwork | sports-science
tags: ["标签1", "标签2"]
publishDate: 2026-04-01
# updateDate: 2026-05-01     # 可选，更新时填写
# draft: true                # 可选，草稿不会发布
---
```

### 分类说明

| category         | 含义     | 示例             |
| ---------------- | -------- | ---------------- |
| `strokes`        | 手法     | 握拍、高远球、吊球 |
| `footwork`       | 步法     | 米字步、启动、上网 |
| `sports-science` | 运动科学 | 热身、发力、伤病   |

### 排版规范

知识库文章自动使用 `KnowledgeLayout`，享有统一的学术排版样式。你只需关注内容本身：

- 使用 `##` 作为主要章节标题
- 使用 `###` 作为子标题
- 使用 `>` 写提示框 / 引用
- 表格使用标准 Markdown 语法
- 代码块用反引号包裹

## 3. 添加教学日记

### 步骤

1. 在 `src/content/lessons/` 下新建 `.mdx` 文件
2. 文件名使用 **kebab-case**（如 `spring-footwork.mdx`）
3. 填写 frontmatter
4. **自由设计页面风格**——可以创建专属 Astro 组件来实现独特的视觉效果

### Frontmatter 模板

```yaml
---
title: "课程标题"
date: 2026-04-01
description: "简短描述"         # 可选，用于列表页预览
tags: ["步法", "热身"]
coverColor: "#F2B5D4"          # 可选，列表页卡片左侧彩条颜色
# draft: true                  # 可选，草稿不会发布
---
```

### 自定义视觉风格

教学日记的核心理念是 **每篇都可以有完全不同的视觉设计**。实现方式：

1. **创建主题组件**：在 `src/components/lessons/` 下创建 Astro 组件（如 `SakuraTheme.astro`）
2. **在 MDX 中导入并使用**：

```mdx
import SakuraTheme from '@/components/lessons/SakuraTheme.astro';

<SakuraTheme>

## 这里是你的内容

正常的 Markdown 语法在这里可以使用。

</SakuraTheme>
```

3. 主题组件负责：字体加载、色彩方案、动画效果、布局结构
4. MDX 负责：内容编写、结构组织

### 注意事项

- MDX 中 HTML 元素和 Markdown 之间需要**空行**分隔
- 在 HTML 块内的文本不会被解析为 Markdown
- `<style is:global>` 可用于组件级全局样式，但需注意不要影响 Nav / Footer
- 使用 `class` 而非 `className`（Astro MDX 使用 HTML 属性风格）

## 5. 发布流程

项目通过 GitHub → Cloudflare Pages 自动部署：

1. 确保本地构建无报错：`npm run build`
2. 提交代码到 `main` 分支
3. 推送到 GitHub：`git push`
4. Cloudflare Pages 会自动触发部署
5. 部署完成后可通过 `shuttle-path.pages.dev` 访问

---

## 6. 常见问题

### 构建报错怎么办？

- 检查 frontmatter 是否缺少必填字段
- 检查 MDX 语法：HTML 和 Markdown 之间是否有空行
- 检查导入路径：使用 `@/` 前缀（如 `@/components/lessons/...`）

### 如何设置文章为草稿？

在 frontmatter 中添加 `draft: true`。草稿在构建时会被过滤，不会出现在网站上。

### 图片怎么放？

目前图片放在 `public/images/` 目录下，在 MDX 中使用相对路径引用：

```md
![描述](/images/your-image.jpg)
```

后续阶段会迁移到 Cloudflare R2 存储。
