## P1 — scale 变化密度过高（"弹簧感"）

- **类**：A
- **状态**：⬜

### 现象
滚一格滚轮画面 scale 可能跳 0.5~1.0（实测 r=0.72→0.75 区间 dScale=-1.00），出现明显的"塌缩 → 猛拉回"的晕车感。

### 技术根因
聚焦段 `dur=3.5`、过渡段 `dur=2`，但二者 `scale` 的 Δ 完全一样（0.7 ↔ 2.2）。过渡段被迫用一半时长走完同样缩放距离，变化速率翻倍。`power3.inOut` 又把 80% 的变化挤在段中间。

### 解决方案
**方案 A（首选）**：把 P/F 的 scale 差收窄，同时延长过渡段
- `P_SCALE` 0.7 → 保持；`F_SCALE` 2.2 → **1.7**
- 过渡段 `dur` 2 → **3**
- 过渡段 scale 的 ease 从 `power3.inOut` 改为 `power2.inOut`（缓一点）

**方案 B（兜底）**：在过渡段中间插一个 "hold" 关键帧，让 scale 在中点保持全景态 0.4~0.6 段时间，避免连续急变。

### 实施办法
`src/pages/index.astro` 改 4 处常量 + ease：
```js
const F_SCALE = isMobile ? 2.0 : 1.7;  // 从 2.6/2.2 降
// segs 中 transition 的 dur: 2 → 3
// addTransition 中 cameraRig 的 ease: 'power3.inOut' → 'power2.inOut'
```

### 验收标准
- 重跑 dScale 采样脚本，最大单步 |dScale| ≤ **0.35**（当前 1.00+）
- 肉眼：从任一聚焦态滚到下一个聚焦态，**不出现"突然变小"的冲击**
- 仍然保留"聚焦 vs 全景"的可辨识差异（不能改到看不出）

### Agent 指示模板
不必委托，直接改参数即可。如果要批量调参→对比效果，命令：
```
/oh-my-claudecode:executor 调整 src/pages/index.astro 的 F_SCALE 到 1.7、过渡段 dur 到 3、过渡段 cameraRig 的 ease 改为 power2.inOut。完成后重启 dev server 并输出 r=0.1~0.3 区间滚动采样的 dScale 峰值。
```

## P2 — 过渡段视觉混乱（4 个 section 半透明共存）

- **类**：C
- **状态**：⬜

### 现象
过渡段所有 `court-row-*` 都是 opacity=0.4，"Shuttle Path / 涨球记录 / 知识库 / 手法 / 步法" 五六个标题同时半透明叠在视口，信息过载。

### 技术根因
`addTransition` 里 `contentSections.forEach(s => tl.to(s, { opacity: 0.4 }))`，没有按距离衰减。

### 解决方案
过渡段 opacity 改为 **0.12**（比聚焦时的非聚焦态 0.15 再低一点），让过渡态唯一可辨识的是"球 + 聚光灯 + 球场线条"，文字几乎隐入背景。

### 实施办法
`addTransition` 里：
```js
tl.to(s, { opacity: 0.12, duration: dur * 0.5, ease: 'power2.inOut' }, pos);
```
同时聚焦段末尾把 hero/lessons/knowledge/cats 都压到 0.12（当前 0.15，已接近，可以一起统一）。

### 验收标准
- 过渡中段（r=0.22、0.48、0.72）截图，能清楚数出：球 + 聚光灯 + 球场白线 + 暗角 —— 文字可见但不抢戏
- 用户肉眼：过渡时"只看到球在飞"，文字是模糊的背景氛围

### Agent 指示
直接改。

## P3 — 聚焦没充满视口

- **类**：B
- **状态**：⬜

### 现象
Hero 聚焦时"Shuttle Path"标题只占视口高度 ~25%，上下大量黑边。Knowledge/Cats 同样问题。

### 技术根因
F_SCALE 2.2 配 rotateX 38° 时，视觉放大率被透视收缩打折；视觉目标（标题）所在位置因 section 几何中心算法偏上，视口利用率低。

### 解决方案
等 P1 收敛后，配合 P12/P13 的聚焦点算法重写一起调。
- F_ANGLE 从 38° 降到 **30°**（俯角变小 → 正面化 → 透视压缩减轻 → 视觉放大率提升）
- F_SCALE 1.7（见 P1）
- 聚焦点从 `offsetHeight * 0.35` 改为**动态读取标题元素实际位置**（见 P13）

### 验收标准
- Hero/Lessons/Knowledge/Cats 聚焦时，目标标题在视口高度的 **35%~55%** 位置
- 标题占视口高度 **≥ 40%**（当前 ~25%）
- 标题四周留白 ≤ 视口高度 30%（非全黑带）

### Agent 指示
和 P12、P13 合并一轮。

## P4 — 退出段 → footer 衔接断层

- **类**：E
- **状态**：⬜

### 现象
退出段结束 rig.scale=0.42，球场缩成屏幕中间一条，上下大片黑；然后 scroll 出 `.scene` 底部，sticky 解除，画面**跳变**。

### 技术根因
`.scene` 500vh + `.camera` 100vh sticky：到达 scene 底部时 camera 从 sticky 变回 static，球场从视口中心掉回文档流本位 → 瞬时跳变。同时 rig 自己还在做 scale 缩小，"双重退场"叠加。

### 解决方案
让"球场离开视口"由 rig 的 y 推出完成，而不是靠 sticky 解除：
- 退出段最后 40% 时长把 `cameraRig.y` 推到 `-window.innerHeight`（球场滑出屏幕上方）
- 同时 footer 用 ScrollTrigger 单独做 fade-in（onLeave 触发）
- `.scene` 底部不再承担转场，sticky 解除时球场已经在屏幕外

### 实施办法
1. `buildAnimation` 里退出段加 `y: -window.innerHeight` 或 `-cameraRig.offsetHeight * 0.6`，ease `power2.in`
2. `BaseLayout.astro` 的 footer 加 `opacity: 0` 默认态 + 一个 `.footer-reveal` class；在 `ScrollTrigger.create` 里监听 scene 末 10% 触发 fade in

### 验收标准
- 从 r=0.95 滚到 r=1.0，**不出现球场瞬时跳位**
- footer 平滑淡入，整个过渡无跳帧

### Agent 指示

## P5 — 球托白圆 vs 场地白线融合

- **类**：D
- **状态**：⬜

### 现象
球托是白色实心圆，落点踩到前发球线、球网、中线时球托消失在白线里，只剩羽毛飘着。

### 技术根因
SVG circle `stroke="white" fill="white" fill-opacity="0.3"` 和场地线颜色一致，缺描边对比。

### 解决方案
给球托加**深色描边外圈 + 内侧亮色**两层结构，不管落到什么背景都有对比度。
- 外层 `stroke="rgba(0,0,0,0.4)" stroke-width="2"`（暗轮廓）
- 内层保持白色填充

或者更简单：整个 shuttle SVG 外面加一层 `filter: drop-shadow(0 0 2px rgba(0,0,0,0.5))` — 但这会引入 P11 的模糊问题，需权衡。

### 实施办法
`src/pages/index.astro` 的 shuttle SVG `<circle>` 加一个兄弟 circle 做描边外圈：
```html
<circle cx="18" cy="38" r="6" stroke="rgba(0,0,0,0.5)" stroke-width="1" fill="none" />
<circle cx="18" cy="38" r="5" stroke="white" stroke-width="1.5" fill="white" fill-opacity="0.3" />
```

### 验收标准
- Cats 聚焦帧（r=0.85）球托与手法/步法中线有清晰视觉分离
- 过渡段球飞过球网时球托仍然可辨

## P6 — 过渡段球飞行轨迹单调

- **类**：C
- **状态**：⬜

### 现象
球只做 y 位移 + scale + rotation ±8°，x 永远锁 50%，像电梯直上直下。doc 15 当初"弧线高远球、最高点在球网"的设计未落地。

### 技术根因
`addTransition` 里 `x: courtX(50)` 写死。

### 解决方案
过渡段 x 做**正弦摆动**：从上一聚焦段 sx → 过中点时偏到对侧 → 下一聚焦段 sx。
- 过渡 1 (Hero 48 → Lessons 56)：中点 x 拉到 **42**
- 过渡 2 (Lessons 56 → Knowledge 43)：中点 x 拉到 **60**
- 过渡 3 (Knowledge 43 → Cats 50)：中点 x 拉到 **40**

同时过渡段 y 在中点时加一个"抬高"效果（模拟高远球最高点），用 `tl.to().to()` 两段：
```js
tl.to(shuttle, { y: midY - 60, x: courtX(midX), ... }, pos);       // 段 A: 升到最高
tl.to(shuttle, { y: nextY, x: courtX(nextX), ... }, pos + dur*0.5); // 段 B: 落到下一站
```

### 实施办法
重写 `addTransition`：接收 `prevSx`、`nextSx`，计算 `midX`、`midY`；拆成两段 keyframe。

### 验收标准
- 过渡段中点截图能看到球明显偏离 50% 中轴
- 球的飞行轨迹在滚动时有"先升后落"的抛物线感

### Agent 指示
```
/oh-my-claudecode:executor 在 src/pages/index.astro 的 addTransition 里把球的飞行轨迹改成两段 keyframe：第一段升到段中点的峰位（y 抬高 60px，x 摆到对侧），第二段落到下一聚焦段。参数：过渡1 midX=42，过渡2 midX=60，过渡3 midX=40。改完给出 r=0.19/0.22/0.25 三帧截图。
```

## P11 — 文字模糊（用户反馈）

- **类**：D
- **状态**：⬜

### 现象
用户肉眼反馈"文字特别模糊"。

### 技术根因（待验证）
候选 3 个：
1. **holo-text 光晕** `text-shadow: 0 0 8px, 0 0 20px, 0 0 40px` 三层叠加在小字号上 → 字边缘被溢出的蓝白光吃掉
2. **translateZ(30px) 浮空**配合 perspective 1200px → 轻微的亚像素渲染不对齐
3. **F_SCALE 2.2 放大** → 文字像素被拉伸（但 holo-text 是白色带光晕，放大应该反而清晰）

改 `src/styles/global.css` 里 `.holo-text`：

```css
.holo-text {
  color: rgba(255, 255, 255, 0.98);  /* 提高主色透明度 */

/* 去掉 ::after 地面投影（如果有） */
```

- 没有"发光模糊"的感觉但保持高级感（用户可接受为"干净白字 + 微光晕"）

## P12 — 聚焦范围因 section 高度不同而大小不一（用户反馈）

- **类**：B
- **状态**：⬜

### 现象
用户反馈"每次落下聚焦到内容标题，范围都或大或小"。

### 技术根因
各 section 的 `min-height` 不同：
- Hero 360px
- Lessons/Knowledge 180px
- Cats 280px
- Footer 48px

但 F_SCALE 固定 1.7（修复后），小 section 被过度放大看起来"太大"，高 section 放大后仍占不满视口看起来"太小"。

### 解决方案
**按 section 高度动态计算 F_SCALE**：
```js
function getFocusScale(section) {
  const targetViewH = window.innerHeight * 0.7;  // 目标：占视口 70%
  const sectionH = section.offsetHeight;
  return Math.min(2.2, Math.max(1.4, targetViewH / sectionH));
}
```
每次 `addFocus` 用 `getFocusScale(seg.section)` 而不是全局 `F_SCALE`。

### 实施办法
1. `addFocus` 函数签名不变，内部算 `fScale = getFocusScale(section)`
2. `cameraRig` 的 scale 目标值用 `fScale` 替换 `F_SCALE`

## P13 — 聚焦落点不精确（用户反馈）

- **类**：B
- **状态**：⬜

### 现象
用户反馈"每次聚焦的范围，都没有完全对准我想要的位置"。

### 技术根因
`getSectionFocusPoint` 用 `offsetTop + offsetHeight * 0.35` 硬算"标题在 section 上部 35%"。实际各 section padding 不同、标题位置也不在 35% 处。

### 解决方案
**每个 section 标一个明确的 focus anchor 元素**（标题本身或特定 div），聚焦点 = 该 anchor 的几何中心。

### 实施办法
1. 在每个 section 的目标标题上加 `data-focus-anchor` 属性：
   ```html
   <h1 class="... holo-text" data-focus-anchor>Shuttle Path</h1>
   ```
2. `getSectionFocusPoint` 改为：
   ```js
   function getSectionFocusPoint(section) {
     const anchor = section.querySelector('[data-focus-anchor]') || section;
     let top = 0;
     let el = anchor;
     while (el && el !== cameraRig) {
       top += el.offsetTop;
       el = el.offsetParent;
     }
     return top + anchor.offsetHeight / 2;
   }
   ```

### 验收标准
- 4 个聚焦段都能让目标标题**在视口高度 45% 左右位置**（垂直居中偏上，符合阅读习惯）
- 用户肉眼确认：聚焦后"眼睛自然落在标题上，不用再找"

## 测试方法论（关于"尽量模拟人的体验"）

### 现状：我能做什么、不能做什么

| 维度 | 能做 | 不能做 |

### 怎么尽量贴近人的体验

既然不能看动画，就用**量化指标反推体感**：

1. **节奏抖动检测**：40 点 dScale 采样 → 看相邻两点最大变化量（本次测试发现峰值 1.00，超出舒适区 3 倍）

3. **性能 trace**：`performance_start_trace` + 滚动回放，看有没有 long task / jank frame（>16ms 帧）
4. **状态交叉验证**：同一 scroll 位置，截图 + 数值采样配对保存，方便后期对照
