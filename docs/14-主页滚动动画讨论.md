# 14 - 主页滚动动画讨论

## 概念

在主页加入 scroll-driven animation（滚动驱动动画），用一颗羽毛球的飞行轨迹串联所有内容区域。

### 用户体验流程

1. 页面加载 → 羽毛球入场动画 → 停在标题/slogan 处
2. 用户开始滚动 → 球沿高远球弧线飞过球场
3. 球经过每个区域 → 该区域的内容显现/高亮

### 技术本质

- **2.5D**：用 2D 技术模拟深度感，不需要真正的 3D 引擎
- 深度线索：近大远小、运动视差、遮挡关系、模糊/淡化、阴影变化
- 核心机制：滚动进度 0%~100% 映射到球在路径上的位置 0%~100%

## 技术方案（初步）

| 方案 | 优点 | 缺点 |
|------|------|------|
| **纯 CSS** (`scroll-timeline`) | 零依赖，原生性能 | 复杂路径控制困难，Safari 支持有限 |
| **GSAP ScrollTrigger** | 业界标杆，路径控制精确，兼容性好 | 需引入 ~30KB 库 |
| **Vanilla JS** | 无依赖 | 手动处理很多细节，开发量大 |

**确认使用 GSAP ScrollTrigger。**

## 设计方案

### 动画节奏（景深切换）

```
滚动进度 ──────────────────────────────────────────────────►

球:    清晰 ────► 模糊      清晰 ────► 模糊      清晰 ──►
内容:  模糊 ────► 清晰      模糊 ────► 清晰      模糊 ──►
       ~~~飞行~~~ [涨球记录] ~~~飞行~~~ [知识库]   ~~~飞行~~~
```

- 飞行阶段：`球 filter:blur(0)` + `内容区 filter:blur(3-4px) + opacity降低`
- 到达阶段：`球 filter:blur(3-4px)` + `内容区 filter:blur(0) + opacity恢复`
- 过渡平滑，用 GSAP 的 ease 控制

### 球的深度变化（俯视视角）

球在空中飞 → 用 scale + shadow 表达高度：
- 飞行中（高空）：scale 较大，阴影偏移大+模糊
- 到达区域（低空）：scale 较小，阴影偏移小+清晰

### 飞行路径

从 Hero 区域出发，沿球场中轴线略带左右摆动的弧线向下飞行，依次经过：
1. Hero（起点）→ 涨球记录 → 网 → 知识库 → 手法/步法 → 运动科学

### 入场动画（非滚动驱动）

页面加载时，球从画面上方飞入 Hero 区域，CSS keyframe 动画，约 0.8s。完成后交给 ScrollTrigger 接管。
