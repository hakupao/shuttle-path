# 07 — 前端设计方案（续）：B+C 混搭 + 去 AI 味 + 手绘风

> 承接 `06-前端设计方向探讨.md`，确定方向为 **Rally（清爽运动）+ Smash（活泼插画）混搭**。
> 本文档深入探讨：字体方案、去 AI 味设计手法、手绘插画风格融入，以及整体视觉语言的落地思路。

---

## 一、设计总纲：确定的方向

### 核心气质
**明亮 · 简洁 · 有力量 · 有温度 · 有手感**

不是冰冷的运动数据面板，也不是可爱到幼稚的卡通风——
是一个 **热爱羽毛球的人手工打造的知识站**，有运动的力量感，也有手绘的温度。

### 关键设计原则
1. **力量与灵巧的对比** — 通过字重粗细、元素大小、线条刚柔来体现
2. **手工感** — 插画线条、有机形状、微瑕纹理，处处透出"人做的"
3. **明亮通透** — 白底为主，彩色点缀，拒绝沉闷
4. **运动节奏** — 版面有快有慢，有紧凑有留白，像一场好的比赛

## 二、字体方案

### 2.1 得意黑（Smiley Sans）— 品牌显示字体

**字体特征：**
- 窄体（Condensed）+ 微倾斜（Oblique）— 天然带运动感和速度感
- 笔画末端带手绘美术字特征 — 契合手绘风主题
- 人文观感与几何特征平衡 — 不呆板、不随意
- 开源免费（SIL OFL），支持中日文 + 拉丁 + 西里尔

**使用场景：**
| 场景 | 用/不用 | 原因 |
|------|---------|------|
| 网站 Logo / 品牌名 | **用** | 辨识度最强的位置 |
| Hero 标语 | **用** | 大尺寸展示最佳效果 |
| 页面大标题（H1） | **用** | 窄体适合大号显示 |
| 文章内标题（H2-H3） | **不用** | 只有一个字重，无法区分层级 |
| 正文 | **不用** | 窄体倾斜不适合长文阅读 |
| 教学日记封面标题 | **用** | 每篇独立设计的点睛之笔 |

**技术方案：**
- 自托管 `SmileySans-Oblique.woff2`（放在 `public/fonts/`）
- `font-display: swap` 避免 FOIT
- 仅用于标题级别，不会影响首屏加载感知

**注意事项：**
- 得意黑只有一个字重，需要搭配其他字体来实现粗细对比
- 它的倾斜是内建的（不是 CSS italic），不需要额外设置 `font-style`

### 2.2 完整字体体系

```
品牌/大标题层：得意黑 Smiley Sans
  └ 用于 Logo、Hero、H1、教学日记封面

内容标题层：Noto Sans SC Bold / Black (700-900)
  └ 用于 H2、H3、卡片标题、强调文字
  └ 粗重的无衬线体 → 力量感

正文层：Noto Sans SC Regular / Medium (400-500)
  └ 用于文章正文、UI 文字
  └ 干净清晰 → 灵巧感

数据/标签层：Inter / JetBrains Mono
  └ 用于数字、标签、技术数据
  └ 西文无衬线 + 等宽 → 精确感
```

### 2.3 字重对比策略 — 力量与灵巧

**核心思路：** 用极端的字重差异制造视觉张力。

```
H1（得意黑）    ██████████████  — 大、窄、倾斜 = 速度 + 力量
H2（Noto 700）  ████████████    — 粗、稳 = 力量
正文（Noto 400） ████            — 轻、匀 = 灵巧
标签（Inter 500） ████ UPPERCASE  — 小、宽字距 = 精准
```

**CSS 实现思路：**
```css
/* 大标题：得意黑，大尺寸，紧凑行高 */
.heading-display {
  font-family: 'Smiley Sans', sans-serif;
  font-size: clamp(2.5rem, 6vw, 5rem);
  line-height: 1.05;
  letter-spacing: -0.01em;
}

/* 内容标题：Noto Sans 粗体，中等尺寸 */
.heading-content {
  font-family: 'Noto Sans SC', sans-serif;
  font-weight: 700;
  font-size: clamp(1.5rem, 3vw, 2rem);
  line-height: 1.25;
}

/* 正文：轻盈，宽松行距 */
.body-text {
  font-family: 'Noto Sans SC', sans-serif;
  font-weight: 400;
  font-size: 1.0625rem;
  line-height: 1.8;
}
```

## 三、去 AI 味设计提案

> 模板和 AI 生成的网站有明显的"味道"：完美对称、均匀间距、标准圆角、
> 渐变蓝紫、三列卡片、stock photo。以下是打破这些模式的具体手法。

### 提案 1：超椭圆（Squircle）替代圆角矩形

**原理：**
CSS `border-radius` 制造的圆角，曲率是突变的 — 从直线突然变成圆弧（G1 连续）。
超椭圆的曲率是渐变的（G2 连续），视觉上更自然、更有机，这正是 Apple 图标看起来"舒服"的原因。

**为什么去 AI 味：**
`border-radius: 12px` 是全世界每个 CSS 框架的默认值。所有模板、所有 AI 生成的界面都用它。
换成超椭圆，即使用户说不出为什么，也会感知到"这个不一样"。

**实现方案：**

```css
/* 方案 A：CSS 原生（Chrome 139+，渐进增强） */
.card {
  border-radius: 20px;
  corner-shape: squircle;  /* 新属性！*/
}

/* 方案 B：带回退 */
.card {
  border-radius: 20px; /* 回退 */
}
@supports (corner-shape: squircle) {
  .card {
    corner-shape: squircle;
  }
}

/* 方案 C：用 SVG clip-path（全浏览器） */
/* 生成超椭圆路径，通过 clip-path 裁切 */
```

**应用层级 — 不同组件用不同曲率：**
```
按钮       → 曲率高（更圆润、可点击感强）
卡片       → 曲率中等（内容容器）
图片容器   → 曲率低（接近矩形但有柔和边缘）
标签/徽章  → 胶囊形（全圆角）
```

### 提案 2：纸质纹理叠层（Grain/Noise Overlay）

**原理：**
在纯色背景上叠一层极淡的噪点纹理，模拟纸张或胶片的颗粒感。

**为什么去 AI 味：**
AI 和模板生成的界面是 "数字完美" 的 — 纯色、无瑕、像塑料。
加一层几乎看不见的噪点，立刻变得"物理"，像印在纸上的东西。

**实现方案（纯 CSS，零依赖）：**
```css
.surface-grain::after {
  content: '';
  position: absolute;
  inset: 0;
  opacity: 0.06; /* 关键：要非常淡，6-10% */
  pointer-events: none;
  background-image: url("data:image/svg+xml,..."); /* SVG feTurbulence */
  mix-blend-mode: overlay;
}
```

**应用场景：**
- Hero 区域背景
- 卡片表面
- 页脚背景
- 教学日记的"纸张"质感

### 提案 3：打破网格对称 — 有机版面

**原理：**
不是所有东西都必须左右等宽、上下等距。故意的不对称创造视觉节奏。

**具体手法：**

**(a) Bento 布局（便当盒）：**
```
┌──────────┬─────┐
│          │     │
│   大卡片  │ 小  │
│          │     │
├────┬─────┴─────┤
│ 小 │           │
│    │   中卡片   │
├────┴──┬────────┤
│       │        │
│  中   │   中   │
└───────┴────────┘
```
不同大小的卡片混搭，每个卡片高度不同，打破"三列等高"的模板感。

**(b) 元素越界：**
```css
/* 图片从卡片里"探出来" */
.card-image {
  margin-top: -2rem;
  margin-left: -1rem;
}

/* 装饰元素不受容器约束 */
.decoration {
  position: absolute;
  right: -20px;
  top: -15px;
}
```

**(c) 不等距间距：**
```css
/* 不要所有 section 都 py-16，创造节奏 */
.section-hero    { padding: 6rem 0 4rem; }  /* 上宽下窄 */
.section-feature { padding: 3rem 0; }        /* 紧凑 */
.section-cta     { padding: 5rem 0 8rem; }   /* 下方留白多 */
```

### 提案 4：手绘线条与 Rough.js

**原理：**
用带随机抖动的线条替代精确的几何线条，模拟手绘效果。

**为什么去 AI 味：**
没有任何 AI 工具会输出"画歪的线"。手绘线条是最直接的"人做的"信号。

**Rough.js — 轻量手绘渲染库（< 9kB）：**
```javascript
import rough from 'roughjs';

// 把一个矩形画成手绘风格
const rc = rough.svg(svgElement);
svgElement.appendChild(
  rc.rectangle(10, 10, 200, 100, {
    roughness: 1.5,    // 手抖程度
    stroke: '#374151',
    strokeWidth: 2
  })
);
```

**应用场景：**
- 分隔线 → 手绘波浪线替代 `<hr>`
- 知识卡片的边框 → 手绘矩形
- 技术讲解的箭头和标注 → 手绘箭头
- 球场示意图的线条 → 手绘风格

**SVG 滤镜方案（更轻量，纯 CSS）：**
```css
/* 给任何元素加上手绘毛边效果 */
.hand-drawn {
  filter: url(#squiggle-filter);
}
```
```html
<svg width="0" height="0">
  <filter id="squiggle-filter">
    <feTurbulence baseFrequency="0.02" numOctaves="3" seed="1" />
    <feDisplacementMap in="SourceGraphic" scale="3" />
  </filter>
</svg>
```

### 提案 5：卡片微旋转 — 散落桌面感

**原理：**
给卡片加上极小的随机旋转角度（< 2°），像照片散落在桌面上。

```css
.card:nth-child(3n+1) { transform: rotate(-0.8deg); }
.card:nth-child(3n+2) { transform: rotate(0.5deg); }
.card:nth-child(3n+3) { transform: rotate(1.2deg); }

/* hover 时归正 */
.card:hover {
  transform: rotate(0deg) translateY(-4px);
  transition: transform 0.3s cubic-bezier(0.34, 1.56, 0.64, 1);
}
```

**适合场景：**
- 教学日记列表（每篇像一张手记卡片）
- 知识库标签云
- 推荐阅读区域

### 提案 6：有机分隔线替代直线

**原理：**
用手绘波浪、曲线、或插画元素替代所有 `<hr>` 和直线分隔。

```
直线（AI 味）：  ────────────────────────
波浪线：         ～～～～～～～～～～～～～
手绘线：         ∿∿ ─ ∿ ── ∿∿∿ ── ∿
插画分隔：       ──── 🏸 ────  (羽毛球相关小插画)
```

**实现：**
- SVG 路径做波浪线，每次随机种子不同
- 或者用几个手绘 SVG 分隔线图案轮流使用

### 提案 7：色彩的非均匀使用

**原理：**
模板网站的配色使用是"均匀"的 — 每个 section 都差不多的色彩分配。
人工设计的网站，色彩有呼吸感 — 大片留白后突然一块亮色，制造节奏。

```
[白白白白白白白白白白白白]  ← 大量留白
[  ████ 亮绿色彩块 ████  ]  ← 突然一块颜色
[白白白白白白白白白白白白]  ← 继续留白
[白 █ 小色块 白白白白白白]  ← 不对称的小色块
[白白白白白白白白白白白白]
```

## 四、手绘插画风格融入方案

### 4.1 插画风格定义

**线条风格：**
- 粗细不均匀的手绘线条（模拟签字笔/马克笔）
- 线条不完全闭合 — 留有缺口，显得松弛
- 偶尔的"画过头" — 线条略微超出交叉点

**色彩风格：**
- 线条本身用深色（石墨灰 #374151 或略带暖色的 #3f3a36）
- 填色用半透明水彩感色块，不追求填满
- 色块故意"涂出线外"一点点

**参考风格（不是照搬）：**
```
线条感    → 类似 MUJI 产品说明书的简笔画
色彩感    → 类似 Procreate 水彩笔刷效果
整体气质  → 球场边白板上教练随手画的战术图
```

### 4.2 插画应用场景

| 场景 | 插画内容 | 实现方式 |
|------|---------|---------|
| 技术动作讲解 | 人物简笔画 + 动作箭头 | 手绘 SVG（可以 AI 辅助生成后微调） |
| 球场示意图 | 手绘风球场 + 站位标记 | Rough.js 动态生成 or 静态 SVG |
| 知识分类图标 | 手绘风：球拍、步法、手型 | SVG 图标集 |
| 页面装饰 | 散落的羽毛球、球拍草图 | 半透明 SVG 点缀 |
| 教学日记封面 | 每篇独立的手绘封面插画 | 自由发挥 |
| 404 页面 | 打飞了的羽毛球 + 搞笑文案 | 手绘插画 |
| 分隔线 | 羽毛球/球拍简笔 | 内联 SVG |

### 4.3 插画产出策略

所有插画/图标素材委托画师或 AI 完成，由开发者提供英文 prompt 和需求说明。

1. **核心图标集**（一次性）— 手绘 10-15 个运动主题图标，全站复用
2. **教学日记封面**（按需）— 每篇课程一张
3. **装饰碎片**（一次性）— 5-6 个散落的羽毛球、球拍碎片 SVG，随机组合装饰
4. **SVG 滤镜**（零边际成本）— 给现有元素加手绘毛边效果（代码实现）

## 五、设计理念借鉴

### 5.1 Bento 布局（便当盒）

不等大的内容块排列，功能性与美感并重。打破"三列等宽"的模板感。

### 5.2 留白的节奏感

"空"不是"没有"，是有意义的留白。
关键不是到处都留白，而是 **在该密的地方密、该空的地方空**，创造呼吸感。

### 5.3 文字作为视觉元素

让标题本身成为画面的一部分 — 超大号标题、文字与图片叠加。
得意黑的窄体倾斜特征非常适合这种处理。

### 5.4 UX 细节的温度

- 顺滑的页面过渡
- 加载状态有温度（不是冰冷的 spinner，而是小动画）

## 六、更新后的配色方案

基于 B+C 混搭 + 手绘风，配色做如下调整：

```
底色层
  白          #FFFFFF    主背景，通透明亮
  暖白        #FEFCF8    知识文章背景，微暖，像好的纸张
  浅灰        #F3F4F6    辅助区块背景

主题色
  活力绿      #10B981    主按钮、链接、活跃状态
  深绿        #059669    hover、强调
  浅绿        #D1FAE5    标签背景、淡化区域

活力色
  阳光橙      #F59E0B    重点标注、教学日记封面色
  珊瑚粉      #FB7185    辅助点缀、热门标签
  柠檬黄      #FCD34D    装饰性色块、高亮

文字色
  石墨        #1F2937    大标题
  深灰        #374151    正文
  中灰        #6B7280    辅助文字
  浅灰        #9CA3AF    占位文字

手绘线条色
  墨线        #3f3a36    手绘插画主线条（略带暖色的深灰）
  淡墨        #9e9890    辅助线条、装饰线
```

**色彩使用策略：**
- 70% 白/浅灰（留白为主）
- 20% 绿色系（品牌一致性）
- 10% 橙/粉/黄（点缀活力）
- 手绘线条始终用墨线色系（跟品牌色系独立，保持手绘感）

## 七、综合提案小结

| 维度 | 方案 |
|------|------|
| **字体** | 得意黑（展示）+ Noto Sans SC（内容）+ Inter（西文），粗细极端对比 |
| **形状** | 超椭圆替代圆角矩形，渐进增强 |
| **纹理** | SVG 噪点叠层，模拟纸张质感 |
| **线条** | Rough.js + SVG 滤镜，手绘风融入全站 |
| **布局** | Bento 不等比网格 + 元素越界 + 不等距间距 |
| **动效** | 弹性缓动 + hover 微旋转归正 + 手绘路径生长动画 |
| **插画** | 核心图标集 + Rough.js 动态生成 + 装饰碎片 |
| **配色** | 白底为主，绿+橙+粉点缀，手绘线独立墨色 |
| **日式** | 便当布局 + 余白 + 文字视觉化 + 季节配色微调 |
