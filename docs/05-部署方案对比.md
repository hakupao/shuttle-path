# Shuttle Path — 部署方案对比：Vercel vs Cloudflare Pages

你问得好。你的域名已经在 Cloudflare 上了，确实应该认真对比一下。

---

## 对比总览

| 维度 | **Vercel (Hobby)** | **Cloudflare Pages (Free)** |
|------|--------------------|-----------------------------|
| **月流量** | 100 GB | **无限** |
| **构建次数** | 无限（6,000 分钟内） | 500 次/月 |
| **单次构建上限** | 45 分钟 | 20 分钟 |
| **并发构建** | 1 | 1 |
| **部署文件数上限** | ~13,000 | 20,000 |
| **单文件大小上限** | 50 MB | 25 MB |
| **边缘节点数** | ~30+（基于 AWS） | **300+（自有网络）** |
| **Preview 部署** | ✅ 每个 PR 自动生成 | ✅ 每个分支自动生成 |
| **Astro 静态站支持** | ✅ 成熟 | ✅ 成熟 |
| **自定义域名** | 需配置 CNAME | **域名在 CF 上，秒级生效** |
| **SSL** | 自动（Let's Encrypt） | 自动（Cloudflare 边缘证书） |
| **媒体存储** | Vercel Blob（有出流量费） | **R2（零出流量费）** |

---

## 逐项分析

### 流量与带宽

- **Cloudflare Pages 胜**：免费无限流量。Vercel 免费 100 GB，对小站够用，但如果你的教学视频自托管并且访问量起来，可能会碰到上限。

### 构建与部署速度

- **Vercel 略胜**：构建缓存更激进，增量构建更成熟。但对于一个 Astro 静态站（几十篇文章），两者差异可忽略，都在 1–2 分钟内。
- Cloudflare 限制 500 次/月构建，日常开发频繁 push 时需注意。但 500 次对一个内容站来说绑绑有余。

### 域名配置

- **Cloudflare Pages 完胜**：你的域名已经在 Cloudflare，选 CF Pages 直接在 dashboard 里点一下就配好了，DNS 无需等待生效。用 Vercel 则需要手动添加 CNAME 记录，等待 DNS 传播。

### CDN 性能

- **Cloudflare Pages 胜**：300+ 个全球节点 vs Vercel 的 ~30 个。对于纯静态站点，Cloudflare 的网络覆盖更广，用户访问更快（尤其是中国周边地区）。

### R2 集成（媒体存储）

- **Cloudflare Pages 完胜**：R2 是 Cloudflare 自家产品，集成天然顺畅，且 **零出流量费**（S3 的最大痛点就是出流量贵）。你的图片、视频、3D 模型都可以放 R2，通过自定义域名访问。
- Vercel 的 Blob 存储有出流量费用，免费额度也更小。

### 开发体验 (DX)

- **Vercel 略胜**：PR 预览集成更精致，部署日志更清晰，dashboard 更现代。但 Cloudflare Pages 近年进步很大，差距已经很小。

### 生态扩展性

- **Cloudflare 胜**：如果未来你想加后端功能（评论、搜索 API 等），Cloudflare 有 Workers、D1 数据库、KV 存储、Queues 等，全在一个平台内闭环。
- Vercel 也有 Serverless Functions，但存储和数据库需要外接服务。

---

## 结论

| 你的情况 | 更适合 |
|----------|--------|
| 域名在 Cloudflare | **CF Pages** ✅ |
| 计划用 R2 存媒体 | **CF Pages** ✅ |
| 纯静态内容站 | 两者都好 |
| 免费无限流量 | **CF Pages** ✅ |
| 开发体验 / PR 预览 | Vercel 略优 |
| 未来可能加后端 | **CF Pages** ✅ |

**建议改用 Cloudflare Pages。**

你的域名已经在 Cloudflare，媒体计划用 R2，网站是纯静态内容站——三个条件都指向 Cloudflare Pages。没有理由绕一圈去 Vercel。
