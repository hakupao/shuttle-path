# Shuttle Path — 技术方案

基于三轮需求讨论的结论，制定以下技术方案。

---

## 1. 技术栈总览

```
┌─────────────────────────────────────────────┐
│               Shuttle Path                   │
├──────────┬──────────┬───────────┬───────────┤
│  Astro   │  React   │    MDX    │ Tailwind  │
│  框架     │  交互组件 │  内容格式  │  样式     │
├──────────┴──────────┴───────────┴───────────┤
│             Cloudflare R2 (媒体)             │
├─────────────────────────────────────────────┤
│      GitHub → Cloudflare Pages (CI/CD)       │
└─────────────────────────────────────────────┘
```

### 选型理由

| 技术 | 作用 | 为什么选它 |
|------|------|-----------|
| **Astro 5** | 站点框架 | 为内容站点而生；默认零 JS、极快；原生 Content Collections 管理知识库；Islands 架构按需加载交互组件 |
| **React 19** | 交互组件 | 教学日记中的定制交互（3D 模型、动画图解、视频播放器等）；生态丰富 |
| **MDX** | 内容格式 | Markdown 写内容 + 嵌入 React 组件；知识库和教学日记都用 MDX |
| **Tailwind CSS 4** | 样式 | 快速开发；响应式天然支持；每节课可以用 CSS 变量覆盖实现不同风格 |
| **Pagefind** | 静态搜索 | 构建时生成索引，零后端；支持中文分词（V2 阶段加入） |
| **Cloudflare R2** | 媒体存储 | 域名已在 CF，R2 免费额度大，零出流量费，视频/图片/3D 模型自托管 |
| **Cloudflare Pages** | 部署 | 域名已在 CF，秒级绑定；无限流量；300+ 全球节点；与 R2 同平台无缝集成（[对比详情](./05-部署方案对比.md)） |

### 结构设计思路

- **知识库** (`content/knowledge/`)：按手法/步法/运动科学分目录，使用统一的 `KnowledgeLayout`，专注内容阅读
- **教学日记** (`content/lessons/`)：每节课一个文件夹，可以包含专属组件和资源。`LessonLayout` 只提供导航和页脚，内部布局完全自由
- **组件分层**：`common/` 全站共享 → `knowledge/` 知识库专用 → 每节课可以有自己的 `components/` 文件夹
- **开发日志**：`docs/` 目录直接作为内容源，在 `/devlog` 路由下渲染

## 3. 内容 Schema

```typescript
// src/content/config.ts
import { defineCollection, z } from 'astro:content';

const knowledge = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string(),
    category: z.enum(['strokes', 'footwork', 'sports-science']),
    difficulty: z.enum(['beginner', 'intermediate', 'advanced']).optional(),
    tags: z.array(z.string()).default([]),
    type: z.enum(['quick-ref', 'tutorial', 'deep-dive']).default('tutorial'),
    relatedTerms: z.array(z.string()).default([]),  // Wiki 互链的关联词条
    draft: z.boolean().default(false),
    publishDate: z.date(),
    updateDate: z.date().optional(),
  }),
});

const lessons = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    date: z.date(),
    description: z.string().optional(),
    tags: z.array(z.string()).default([]),
    coverImage: z.string().optional(),      // 封面图（用于卡片展示）
    coverColor: z.string().optional(),      // 封面色（用于卡片背景）
    draft: z.boolean().default(false),
  }),
});

export const collections = { knowledge, lessons };
```

## 4. 关键功能实现

### 4.1 知识互链 `<Term>` 组件

```tsx
// src/components/common/Term.tsx
// 将专有名词自动链接到知识库对应词条
// 鼠标悬停显示摘要预览卡片

<Term>高远球</Term>
// → 渲染为带下划线的链接，指向 /knowledge/strokes/clear
// → hover 时弹出摘要卡片
```

实现思路：
1. 构建时生成一份「术语 → URL + 摘要」的映射表
2. `<Term>` 组件查找映射表，渲染链接 + Popover

### 4.2 教学日记的自由布局

每节课的 MDX 可以完全自定义：

```mdx
---
title: "第一课：握拍与发球"
date: 2026-04-03
coverColor: "#2d5a3d"
---

import CustomHero from './components/CustomHero'
import { motion } from 'framer-motion'

<CustomHero />

<motion.div initial={{ opacity: 0 }} animate={{ opacity: 1 }}>
  今天的内容是...
</motion.div>
```

`LessonLayout` 只负责渲染导航和页脚，中间区域完全交给 MDX 内容控制。

### 4.3 媒体资源管理

```
Cloudflare R2 Bucket
├── images/
│   ├── knowledge/      # 知识库配图
│   └── lessons/        # 教学日记配图
├── videos/             # 教学视频
└── models/             # 3D 模型文件（后期）

访问方式：https://media.你的域名.com/images/...
```

在 Astro 中配置一个媒体 URL 前缀，引用时只需写相对路径。

## 5. 部署架构

> 方案对比详情见 [05-部署方案对比.md](./05-部署方案对比.md)

```
你写内容 (MDX)
    ↓
推送到 GitHub
    ↓
Cloudflare Pages 自动构建 (Astro build)
    ↓
部署到 Cloudflare 全球边缘网络 (300+ 节点)
    ↓
用户通过你的域名访问（域名已在 CF，零配置）
```

配置步骤（搭项目时执行）：
1. GitHub 创建仓库 `shuttle-path`
2. Cloudflare Pages 导入 GitHub 仓库，自动检测 Astro
3. 绑定自定义域名（域名已在 CF，秒级生效）
4. 配置 R2 存储桶用于媒体资源（Phase 2）

## 6. 开发阶段规划

### Phase 1: MVP（当前）

搭建项目骨架，上线最小可用版本。

| 任务 | 内容 |
|------|------|
| 项目初始化 | Astro + React + MDX + Tailwind 搭建 |
| 基础布局 | 全站导航、页脚、响应式框架 |
| 首页 | Hero + 内容入口卡片 + 最新教学日记 |
| 知识库 | KnowledgeLayout + 3 篇初始文章（握拍、高远球、基本站位） |
| 教学日记 | LessonLayout + 1 篇示范日记 |
| 开发日志 | 渲染 docs/ 目录 |
| 部署 | GitHub → Cloudflare Pages，域名绑定 |

### Phase 2: 内容增强

| 任务 | 内容 |
|------|------|
| `<Term>` 组件 | 知识互链 + 悬停预览 |
| 标签系统 | 标签页面、标签筛选 |
| Pagefind 搜索 | 全站搜索（支持中文） |
| 媒体迁移 | 配置 Cloudflare R2，媒体资源上 CDN |
| VideoPlayer | 自定义视频播放组件 |

### Phase 3: 高级功能

| 任务 | 内容 |
|------|------|
| 3D 模型查看器 | Three.js / React Three Fiber 渲染发力分析 |
| 知识图谱可视化 | 节点连线图展示知识关系 |
| Giscus 评论 | 基于 GitHub Discussions |
| SEO 优化 | sitemap、og:image、结构化数据 |
| 性能优化 | 图片懒加载、视频预加载策略 |
