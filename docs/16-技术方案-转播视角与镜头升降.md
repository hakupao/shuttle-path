## 一、DOM 结构改造

在 `<main>` 和现有 `.court-outer` 之间插入三层包装：

```html
<main class="flex-1 pt-16">
  <div class="scene">             <!-- 1. 滚动空间容器 -->
    <div class="camera" id="camera">         <!-- 2. 摄像机 -->
      <div class="camera-rig" id="camera-rig">  <!-- 3. 摄像机云台 -->

        <div class="court-outer">   <!-- 原有结构不变 -->
          <div class="court" role="main">
            ...
          </div>
        </div>

      </div>
    </div>
  </div>
</main>
```

### 各层分工

| 层 | CSS 职责 | GSAP 职责 |
|----|---------|-----------|
| `.scene` | `height: 700vh`，提供滚动空间 | ScrollTrigger 的 `trigger` |
| `.camera` | `perspective`, `perspective-origin`, `overflow: hidden` | ScrollTrigger 的 `pin` 目标，固定在视口中 |
| `.camera-rig` | `transform-style: preserve-3d` | 接收 `rotateX` / `scale` / `translateY` 动画 |
| `.court` | 移除 `overflow: hidden`，加 `transform-style: preserve-3d` | 子元素用 `translateZ` 浮空 |

### 为什么要这样分

1. **CSS 规范要求**：`perspective` 必须在父元素，3D 变换在子元素，才能正确投影。所以 perspective 放 `.camera`，变换放 `.camera-rig`。
2. **pin 需要稳定容器**：ScrollTrigger 的 pin 会给元素加 `position: fixed`，放在 `.camera` 上可以让内部 3D 变换不受干扰。
3. **滚动距离由 scene 控制**：700vh 给 4 次聚焦 + 3 次过渡 + 入场退场提供充足空间。

### overflow:hidden 的迁移

`.court` 当前有 `overflow: hidden`，但它和 `transform-style: preserve-3d` **冲突**（浏览器会强制降级为 flat 渲染）。

解决办法：
- 从 `.court` **移除** `overflow: hidden`
- 改到 `.camera` 层做视口裁剪
- `.court` 的 `border-radius: 2px` 仍然提供视觉边界
- 噪点纹理（`::before`）的 `inset: 0` 天然被父元素约束，不需要 overflow

## 二、CSS 3D 基础样式

```css
/* 滚动空间 */
.scene {
  height: 700vh;
  position: relative;
}

/* 摄像机 — 提供透视投影 */
.camera {
  width: 100%;
  height: 100vh;
  display: flex;
  align-items: center;
  justify-content: center;
  perspective: 1200px;
  perspective-origin: 50% 85%;   /* 摄像机在球场底端后方 */
  overflow: hidden;
}

/* 摄像机云台 — GSAP 动画的目标 */
.camera-rig {
  transform-style: preserve-3d;
  transform: rotateX(25deg) scale(0.55);  /* 初始状态：全景视角 */
  will-change: transform;
}

/* 球场本体 — 允许子元素 3D 浮空 */
.court {
  /* 移除 overflow: hidden */
  transform-style: preserve-3d;
}
```

### 参数解释

| 参数 | 值 | 说明 |
|------|----|------|
| `perspective` | `1200px` | 中等透视强度，不会太夸张也不会太平 |
| `perspective-origin` | `50% 85%` | 模拟转播机位：球场底端后方偏上 |
| 初始 `rotateX` | `25deg` | 全景状态的俯角，产生近大远小的梯形 |
| 初始 `scale` | `0.55` | 让整个球场缩小到视口内可见 |

## 三、GSAP 时间线重构

### 滚动节奏

不再使用第一阶段的 7 个均匀航点。改为**聚焦/过渡交替**结构：

```
时间线:  [入场] [聚焦Hero] [过渡↑] [聚焦课程] [过渡↑] [聚焦知识库] [过渡↑] [聚焦分类] [退场]
权重:     1.5      4        2        4        2        4          2        4      1.5
```

- **聚焦段**（duration=4）：镜头俯冲放大，内容充满视口，滚动距离长，用户可以从容阅读
- **过渡段**（duration=2）：镜头拉起缩小，球飞行，滚动距离短，快速飞越
- 总权重 ~25，映射到 700vh 的滚动距离

### ScrollTrigger 配置

```javascript
const tl = gsap.timeline({
  scrollTrigger: {
    trigger: '.scene',         // 触发器改为 scene（700vh 高度）
    pin: '#camera',            // pin 摄像机，固定在视口
    start: 'top top',
    end: 'bottom bottom',
    scrub: 1.5,
    pinType: 'transform',      // iOS Safari 用 transform 替代 fixed，避免抖动
    invalidateOnRefresh: true,
  },
});
```

### 聚焦段动画（以"聚焦课程区"为例）

GSAP 在同一时间点并行动画多个目标：

```javascript
const pos = tl.duration();  // 当前时间线位置

// 1. 摄像机云台 — 俯冲 + 放大 + 平移居中
tl.to('#camera-rig', {
  rotateX: 48,              // 俯角增大 → 更接近正上方
  scale: 2.8,               // 放大 → 内容充满视口
  y: -sectionCenterY,       // 向上平移 → 目标区域居中
  duration: 4,
  ease: 'power2.inOut',
}, pos);

// 2. 羽毛球 — 降落
tl.to('#shuttle', {
  top: targetY,
  left: targetX + '%',
  scale: 0.75,
  rotation: 0,
  duration: 4,
  ease: 'power3.out',
}, pos);

// 3. 聚光灯 — 收拢
tl.to('#spotlight', {
  top: targetY,
  left: targetX + '%',
  scale: 0.7,
  opacity: 0.9,
  duration: 4,
  ease: 'power3.out',
}, pos);

// 4. 内容区 — 聚焦目标 opacity:1，其余 opacity:0.3
contentSections.forEach(section => {
  tl.to(section, {
    opacity: section === targetSection ? 1 : 0.3,
    duration: 4,
    ease: 'power2.out',
  }, pos);
});
```

### 过渡段动画

```javascript
const pos = tl.duration();

// 1. 摄像机云台 — 拉起 + 缩小 → 全景
tl.to('#camera-rig', {
  rotateX: 22,              // 俯角减小 → 更平视，纵深感更强
  scale: 0.55,              // 缩小 → 看到整个球场
  y: 0,                     // 回到中心
  duration: 2,
  ease: 'power2.inOut',
}, pos);

// 2. 羽毛球 — 飞行
tl.to('#shuttle', {
  top: midpointY,
  left: midpointX + '%',
  scale: 1.3,
  rotation: 8,
  duration: 2,
  ease: 'power2.inOut',
}, pos);

// 3. 聚光灯 — 扩散
tl.to('#spotlight', {
  scale: 1.4,
  opacity: 0.5,
  duration: 2,
  ease: 'power2.inOut',
}, pos);

// 4. 所有内容区统一半透明
contentSections.forEach(section => {
  tl.to(section, {
    opacity: 0.5,
    duration: 2,
    ease: 'power2.inOut',
  }, pos);
});
```

### translateY 的计算方式

聚焦某区域时，需要把它居中到视口：

```javascript
function getSectionCenterY(section) {
  const rigCenter = cameraRig.offsetHeight / 2;
  const sectionCenter = section.offsetTop + section.offsetHeight / 2;
  return sectionCenter - rigCenter;  // 正值 = 向上移
}
```

> 注意：在 `rotateX` 透视下，translateY 的视觉效果会受前后位置影响（近处移动看起来更大）。具体数值需要实际调试。

## 四、全息文字

### 样式（新增到 global.css）

```css
/* 全息文字 — 干净风格 */
.holo-text {
  color: rgba(255, 255, 255, 0.95);
  text-shadow:
    0 0 8px rgba(200, 220, 255, 0.4),     /* 近距离蓝白光晕 */
    0 0 20px rgba(180, 210, 255, 0.15),    /* 中距离扩散 */
    0 0 40px rgba(160, 200, 255, 0.05);    /* 远距离微光 */
}

/* 地面投影 — 暗示悬浮高度 */
.holo-text::after {
  content: '';
  position: absolute;
  bottom: -6px;
  left: 10%;
  right: 10%;
  height: 4px;
  background: rgba(0, 0, 0, 0.15);
  filter: blur(4px);
  border-radius: 50%;
}
```

### 3D 浮空

内容区域加 `translateZ` 脱离球场平面：

```css
.court-row {
  transform: translateZ(30px);   /* 浮在球场上方 30px */
}
```

- `translateZ(30px)` 在 `perspective: 1200px` 下对大小几乎无影响（30/1200 ≈ 2.5%）
- 但在 `rotateX` 倾斜时会产生明显的层次分离：球场线条在 Z=0，文字在 Z=30
- 配合地面投影，形成"悬浮记分牌"的感觉

## 五、性能保障

### 核心原则：只动画 GPU 友好属性

| 元素 | 动画属性 | 备注 |
|------|---------|------|
| camera-rig | `transform`（rotateX, scale, translateY） | 单一 composite layer |
| shuttle | `transform`（translate, scale, rotation） | 已有 will-change |
| spotlight | `transform` + `opacity` | 已有 will-change |
| 内容区 | `opacity` | 不使用 blur（与 preserve-3d 冲突） |

### 景深方案（已确认用 opacity 替代 blur）

- 聚焦区域：`opacity: 1`
- 非聚焦区域：`opacity: 0.3`
- 飞行过渡中：全部 `opacity: 0.5`
- 靠聚光灯 + 透明度差异引导视觉焦点

### 降级策略

如果帧率低于 60fps，逐步降级：
1. 移除 `text-shadow` 外层（减少合成开销）
2. 简化灯光（减少 radial-gradient 数量）
3. 降低 perspective 精度

### 无障碍

```javascript
if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
  // 跳过所有 3D 效果，直接显示静态布局
  return;
}
```

## 六、移动端适配

### 响应式参数

```css
/* 手机默认 */
.camera {
  perspective: 800px;     /* 小屏用较小 perspective */
}
.camera-rig {
  transform: rotateX(22deg) scale(0.5);
}

/* 平板及以上 */
@media (min-width: 768px) {
  .camera { perspective: 1200px; }
  .camera-rig { transform: rotateX(25deg) scale(0.55); }
}
```

### 聚焦缩放比例

屏幕越小，聚焦时需要更大的 scale 才能让内容填满视口：

| 设备 | 聚焦 scale | 全景 scale |
|------|-----------|-----------|
| 手机（<768px） | ~3.2 | 0.5 |
| 平板（768px+） | ~2.8 | 0.55 |
| 桌面（1024px+） | ~2.5 | 0.55 |

用 `ScrollTrigger.matchMedia` 按断点配置不同参数。

## 八、风险与应对

| 风险 | 说明 | 应对方案 |
|------|------|---------|
| `preserve-3d` 与 `overflow:hidden` 冲突 | 3D 子元素会被降级为 flat | overflow 迁移到 .camera 层 |
| 透视下点击区域偏移 | 链接位置和视觉位置不一致 | 聚焦状态接近正面（rotateX ~48°），偏移小；过渡状态可加大点击区域 |
| iOS Safari pin 抖动 | 手机滚动时 fixed 定位卡顿 | 使用 `pinType: 'transform'` |
| 文字在倾斜视角下变形拉伸 | 阅读困难 | 聚焦时俯角增大到 ~48°（接近正面），文字变形小 |
| `box-shadow` 在 3D 变换下异常 | 球场灯光溢出效果错位 | 把 box-shadow 移到不参与 3D 变换的兄弟元素上 |
| 700vh 滚动中过渡段视觉空洞 | 用户觉得无聊 | 过渡段加入球场线条/球网的微动画 |
