# Shuttle Path — 需求讨论

## 项目概述

一个羽毛球教学知识平台，集知识库、备课日记、讲义、教学笔记于一体。
目标：好看、好用，服务于自己和学员。

### Q2: 内容管理

你希望怎么写内容？
- a) 在网站后台编辑器里写
- b) 用 Markdown 文件写，网站自动渲染
- c) 两者都要

**A2:**
我不知道有什么方案可以使用，但我比较喜欢展现形式的丰富多样（图+文、视频、表格等
知识库可以做成比较学术风的风格
但每次的教学日记，学习记录，我都想用完全不同的前端风格呈现
请你给我建议

> **💡 建议：使用 MDX（Markdown + 组件）**
>
> MDX 是 Markdown 的超集，你可以在 Markdown 中直接使用前端组件。这意味着：
>
> - **知识库文章**：用标准 Markdown 写，统一学术风模板，专注内容
> - **教学日记**：每篇可以引入不同的布局组件、配色、动画，实现 pudding.cool 那种"每期风格不同"的效果
> - **富媒体**：图文、视频嵌入、交互式表格、动画图解都能支持
>
> 写作体验：你在 VS Code 或任何编辑器里写 `.mdx` 文件，push 到 GitHub 就自动发布。
>
> ```mdx
> ---
> title: "第三课：正手高远球"
> date: 2026-04-01
> # 每节课可以自由引入不同组件和样式，不走模板
> ---
>
> import VideoPlayer from '@/components/VideoPlayer'
> import MuscleMap from '@/components/MuscleMap'
>
> 今天我们来学习正手高远球的发力原理。
>
> <VideoPlayer src="/videos/lesson-03-demo.mp4" />
>
> ## 发力链分析
>
> <MuscleMap highlight={['三角肌', '背阔肌', '前臂']} />
> ```

### Q4: 技术偏好

你有前端/后端开发经验吗？对技术栈有偏好吗？（比如 React、Vue，或者不确定都行）

**A4:**
有开发经验，但都是vibe coding
所以希望你可以根据我的需求和想法，给我一些技术上的建议和方案
我的域名是在cloudflare
所以我希望这个项目将来推送到github上后，可以自动部署（不局限于cf，也可以上vercel等平台）
并且根文件夹的我们讨论的过程，也可以完全公开，作为这个项目的开发日志，记录我们讨论的过程和技术方案的选择

> **💡 技术方案建议：Astro + React + MDX**
>
> | 选型 | 理由 |
> |------|------|
> | **Astro** (框架) | 专为内容站点设计，默认零 JS、极快；原生支持 MDX 和内容集合；每个页面可以按需引入 React 组件 |
> | **React** (组件层) | 用于制作每节课的交互式视觉组件（动画、图表、肌肉图等） |
> | **MDX** (内容格式) | Markdown 写内容 + 嵌入组件，兼顾写作效率和展现自由度 |
> | **Tailwind CSS** (样式) | 快速打磨细节，配合每节课的自定义主题变量 |
> | **GitHub + Cloudflare Pages** (部署) | push 即部署；域名已在 CF，秒级绑定；无限流量；与 R2 同平台集成 |
>
> **为什么选 Astro 而不是 Next.js？**
> - 你的网站核心是**内容展示**，不是 Web App
> - Astro 默认输出纯 HTML，只在需要交互的地方才加载 JS，性能更好
> - Astro 的 Content Collections 天然适合管理知识库和教学日记
> - 同样支持 React 组件，不牺牲交互能力
>
> **项目结构预览：**
> ```
> shuttle-path/
> ├── docs/                    ← 开发日志（我们的讨论，公开）
> │   └── 01-需求讨论.md
> ├── src/
> │   ├── content/
> │   │   ├── knowledge/       ← 知识库文章（学术风，统一模板）
> │   │   └── lessons/         ← 教学日记（每篇可有独特风格）
> │   ├── components/          ← 可复用组件（视频播放器、肌肉图等）
> │   ├── layouts/             ← 页面布局模板
> │   │   ├── KnowledgeLayout.astro   ← 知识库统一布局
> │   │   └── lessons/         ← 每节课可以有自己的布局
> │   └── pages/               ← 路由页面
> ├── public/                  ← 静态资源（图片、视频）
> ├── astro.config.mjs
> └── package.json
> ```

### Q5: 内容结构

你心目中的知识库大概怎么分类？比如：
- 按技术分：发球、步法、击球、战术……
- 按阶段分：入门、进阶、高级……
- 按运动学分：肌肉、发力、损伤预防……
- 其他想法？

**A5:**
羽毛球分两大类，手法和步伐
同时我还想科普一些运动学的知识，帮助学员更好地理解羽毛球的动作原理
热身，放松，发力技巧，伤病预防，肌群锻炼等等我都想慢慢扩展
但我并不了解国际上对于羽毛球教学的分类体系是什么样的，所以我希望你可以给我一些建议，帮我设计一个合理的内容结构

> **💡 内容结构建议**
>
> 参考 BWF（世界羽联）教练认证体系和常见的运动教学分类法，建议如下：
>
> ```
> 知识库
> ├── 🏸 手法 (Strokes & Techniques)
> │   ├── 握拍基础
> │   ├── 发球（正手/反手/高远/短球）
> │   ├── 后场技术（高远球、吊球、杀球、被动球）
> │   ├── 前场技术（搓球、推球、勾球、扑球）
> │   ├── 中场技术（平抽挡、接杀）
> │   └── 双打专项（封网、轮转配合）
> │
> ├── 👣 步法 (Footwork)
> │   ├── 基本站位与启动（split step）
> │   ├── 上网步法
> │   ├── 后退步法
> │   ├── 两侧移动
> │   ├── 全场串联
> │   └── 双打站位与轮转
> │
> ├── 🧠 战术 (Tactics)  ← 可以后期扩展
> │   ├── 单打战术
> │   └── 双打战术
> │
> └── 🏋️ 运动科学 (Sports Science)
>     ├── 热身与放松
>     ├── 发力原理（动力链、鞭打发力）
>     ├── 体能训练（核心、下肢、爆发力）
>     ├── 伤病预防（膝盖、肩袖、脚踝、手腕）
>     └── 恢复与营养
> ```
>
> **设计思路：**
> - 手法和步法是主干，和你的教学习惯一致
> - 运动科学作为横向知识，可以在手法/步法文章中交叉引用（比如"高远球"文章里链接到"鞭打发力原理"）
> - 战术板块留空，后期需要时扩展
> - 每篇文章可以标注难度等级（入门/进阶/高级），方便学员筛选
>
> **❓ 追问：这个分类结构你觉得合理吗？有想调整的地方吗？**

### Q7: 设计风格

你喜欢什么风格的网站？可以描述感觉，也可以举你喜欢的网站例子。
- 简洁现代？
- 运动感强烈？
- 温暖亲和？
- 其他？

**A7:**
风格简洁现代即可，但我对细节的打磨很看重
https://pudding.cool/ 这个网站，每一期的内容风格都不一样，但每一期的细节都很精致，给人一种很用心的感觉
我的每节教学课就像参考这个风格（他们内部点开，外面的导航不要这样）
你有什么建议吗
我觉得可以调用一些前端设计的 skill

> **💡 设计方案建议**
>
> Pudding.cool 的核心理念：**统一的外壳 + 独立的内核**。翻译到你的项目：
>
> | 层级 | 风格策略 |
> |------|----------|
> | **全站导航/首页** | 简洁现代，统一的导航栏、一致的品牌感 |
> | **知识库页面** | 统一的学术风模板，干净的排版，专注阅读 |
> | **教学日记页面** | 每节课独立的视觉主题——不同的配色、布局、动画、插图风格 |
>
> **实现方式：**
> - 每节课的 MDX frontmatter 里定义 `theme` 字段（配色、背景、字体风格）
> - 建立一套"课程主题"系统：预设几个基础主题（海洋、森林、日落、水墨……），也可以单独定制
> - 全站导航始终保持一致，进入具体课程后切换到该课的视觉风格
>
> **关于细节打磨：**
> 可以逐步积累的精致细节：
> - 页面滚动时的微动画（内容渐入、视差效果）
> - 图文交错的呼吸感排版
> - 视频/GIF 动作演示的精心裁剪
> - 移动端的阅读体验优化
>
> 等进入具体的前端设计阶段，我们可以逐页打磨。
