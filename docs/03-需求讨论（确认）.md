### Q18: 教学日记的视觉主题

你提到每节课想用不同的前端风格。你现在心目中有没有一些想象？比如：
- 某节课是清新绿色调配合手绘风插图
- 某节课是深色系配合运动摄影大图
- 某节课是水墨风

还是说你希望我来设计一套可选的主题系统，你写每节课时从中挑选？

**A18:**
每次课程编写时，我都会打开脑洞与你讨论，或者我给你我看到的网站让你复刻
不需要模板

> **💡 回复：**
>
> 明白了！每节课都是**独立的创作**，不走模板路线。这和 pudding.cool 的理念完全一致——每篇都是定制品。
>
> 技术上的影响：
> - 不做主题模板系统，每节课的 MDX 文件可以自由引入任何自定义组件和样式
> - 架构上只需要保证：**全站导航一致** + **每节课内部完全自由**
> - 每次写新课时，你给我灵感（描述 / 参考网站链接），我来实现前端效果
>
> 这样每节课都是一次小型的前端创作，很有意思。

### Q19: 知识库文章的深度

知识库里的文章你打算写多深？
- a) 简明扼要，像速查卡片（200–500字）
- b) 中等深度的教程（500–2000字，配图配视频）
- c) 深入详解（长文，运动学原理 + 动作分析 + 练习方法）
- d) 不同文章深度不同

**A19:**
d

> ✅ **结论：** 文章深度按需而定。模板需要灵活——既能承载短小的速查卡片，也能承载带视频和图解的长文。
>
> 实现方式：在 frontmatter 中标注 `type: quick-ref | tutorial | deep-dive`，不同类型可以有不同的排版样式。

## 三轮讨论总结

| 项目 | 结论 |
|------|------|
| **定位** | 羽毛球教学知识平台（知识库 + 教学日记） |
| **用户** | 公开站点，无需登录 |
| **技术栈** | Astro + React + MDX + Tailwind CSS |
| **部署** | GitHub → Cloudflare Pages（域名已在 CF，秒级绑定） |
| **内容格式** | MDX（Markdown + 组件），支持图文/视频/3D |
| **知识库分类** | 手法 / 步法 / 运动科学（参考 BWF 体系） |
| **知识互链** | Wiki 式 `<Term>` 组件，专有名词互相链接 |
| **教学日记** | 每篇可有独立视觉主题，公开展示 |
| **媒体** | Cloudflare R2 托管，自托管视频 |
| **语言** | 中文 |
| **搜索** | 导航 + 标签 + Pagefind 搜索（V2） |
| **互动** | V1 无互动，后期可加 Giscus 评论 |
| **响应式** | 移动端 + 桌面端都做好 |
| **关于作者** | 低调融入 footer，链接 GitHub |
| **上线策略** | MVP 先行，快速迭代 |
